# 航班选择路由规则

## 决策树

```
输入：出发城市、目的城市、出发日期
    ↓
[1] 查询直飞航班
    ├─ 有直飞 → 筛选经济舱 → 按价格+时间排序 → 推荐 Top 3
    └─ 无直飞 → 进入 [2]
    ↓
[2] 查询经停航班（同一航班号、同一张票）
    ├─ 识别经停机场和停留时间
    ├─ 有经停 → 筛选经济舱 → 按总价+总时长排序 → 推荐 Top 3
    └─ 无经停 → 进入 [3]
    ↓
[3] 查询联程航班
    ├─ 设置中转停留窗口：同机场 1.5-2 小时，跨机场 2 小时内
    ├─ 有联程 → 筛选经济舱 → 按总价+总时长排序 → 推荐 Top 3
    └─ 无联程 → 进入 [4]
    ↓
[4] 查询临近城市方案
    ├─ 查找目的城市周边高铁 1 小时内可达的城市
    ├─ 查询出发城市 → 临近城市的航班
    ├─ 查询临近城市 → 目的城市的高铁
    └─ 组合方案 → 按总价+总时长排序 → 推荐 Top 3
```

## 排序权重

| 因素 | 权重 | 说明 |
|------|------|------|
| 方案类型 | 25% | 直飞 > 经停 > 联程 > 临近城市接驳 |
| 总价格 | 18% | 经济舱票价 |
| 总时长 | 25% | 含经停/中转等待时间 |
| 出发时间 | 10% | 匹配用户偏好（早/午/晚） |
| 航空公司 | 22% | 用户偏好东方航空/上海航空，其次优先大型航司 |

## 用户航司偏好

- 东方航空（MU）和上海航空（FM）为优先航司。
- 同等可执行性下，优先推荐 MU/FM，即使票价略高或总耗时略长。
- 如 MU/FM 方案导致无法满足拜访时间、换乘风险明显过高，或成本显著高于其他方案，应推荐更可执行方案，并说明未采用 MU/FM 的原因。

## 无锡出发的上海虹桥接驳规则

当出发城市为无锡时，除查询无锡硕放机场航班外，还应查询：

1. 无锡东 -> 上海虹桥高铁。
2. 上海虹桥机场 T2 出发的东方航空/上海航空航班；必须优先使用精确机场对 `SHA -> 目的地机场` 或具体航班号补查，不能只依赖城市级“上海 -> 目的地”的聚合结果。
3. 高铁抵达上海虹桥站到虹桥机场 T2 的衔接时间。

虹桥站到虹桥机场 T2 的用户经验参数：

| 环节 | 估算时间 |
|------|---------|
| 上海虹桥高铁站出站口 -> 虹桥机场 T2 | 15 分钟 |
| 虹桥机场 T2 安检 | 20 分钟 |
| 基础换乘耗时 | 35 分钟 |

排程约束：

- 不托运行李且已线上值机：高铁抵达上海虹桥站到航班起飞至少预留 60 分钟。
- 需要托运行李：高铁抵达上海虹桥站到航班起飞至少预留 90 分钟。
- 节假日、早晚高峰或航司柜台排队明显时，应增加额外缓冲。

偏好航司漏查防护：

- 若城市级航班查询只返回浦东（PVG）或非 MU/FM 航班，必须继续查上海虹桥（SHA）到目的地机场。
- 若用户或历史经验指出具体偏好航班号（如 MU/FM），必须按航班号再次核验该航班当天是否执飞。
- 偏好航司方案即使不作为首推，也必须进入交通方案对比或“偏好航司核验”段落。

## 临近城市判断标准

- 高铁行程 ≤ 1 小时
- 有直达高铁（无需换乘）
- 高铁班次 ≥ 5 班/天

## 中转时间要求

硬过滤规则：凡是经停停留或联程中转停留超过 120 分钟的航班，不进入推荐列表。除非用户明确要求“便宜优先、可以长时间中转”，否则不要展示为备选。

| 场景 | 最短 | 最长 |
|------|------|------|
| 经停同票 | 30 分钟 | 90 分钟 |
| 同机场中转 | 1.5 小时 | 2 小时 |
| 跨机场中转 | 2 小时 | 2 小时 |

## 特殊规则

- 早班机（6:00-9:00）：需确认出发地到机场的交通时间
- 晚班机（20:00-23:00）：需确认到达后到酒店的交通方式
- 红眼航班（23:00-6:00）：不推荐，除非用户明确要求
